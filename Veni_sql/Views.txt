
	Views
	---------
	A View is a virtual table that is created based on the result of a SQL query.

        A)It does not store data itself(except special cases)
        B)It displays data from one or more underlying tables.
	
	Ex:
	Create view vw_emp
	AS
	SELECT EMPNO,ENAME,DEPARTMENT FROM EMPLOYEES
	
	Note: We can use it like a table
             SELECT * FROM vw_emp
       
        Why Do We need Views:
        ====================
	1)Security-Restrict access to specific columns or rows(users cant see salary/sensitive data
	2)Simplify Complex Joins - Hide complex joins or logic
	3)Data Abstraction - Users don't need to know underlying table structure
	4)Reusability - Write once, reuse multiple times
	5)Consistency - Same logic used across reports/applications
	
        Ex:
        CREATE VIEW vw_empdetails
        AS
        SELECT NAME,DEPARTMENT FROM EMPLOYEES
        Note:Users can not see salary data

        Ex:
        CREATE VIEW VW_EMPDEPT
        AS 
        SELECT EMPID,ENAME,SAL FROM EMPLOYEE E
	JOIN DEPARTMENT AS D
	ON E.DNO=D.DNO

	Types of Views:
	==============
	1)Simple View
          a)Based on one table
	  b)No aggregate functions, GROUP BY etc.
	  c)Usually updatable

        Ex:
	CREATE VIEW VW_EMPLOYEE
	AS
	SELECT EMPID,ENAME,SALARY FROM EMPLOYEE

        UPDATE VW_EMPLOYEE
	SET SALARY=50000
        WHERE EMPID=1
     
	2)Complex View
	 a)Based on multiple tables
	 b)May include: 
			JOIN
			GROUP BY
			AGGREGATE FUNCTIONS(SUM,COUNT ETC.)
	EX:
	CREATE VIEW vw_Deptsal
	AS
	SELECT DNO,SUM(SALARY) AS TOTALSALRY FROM EMPLOYEES
        GROUP BY DNO

	3)Indexed View / Materialized view
	 a)Physically stores data
	 b)Improves performance
	 c)Requires a unique clustered index
	
	Ex:
	Create view vw_Totalsal
	WITH SCHEMABINDING
	AS
	SELECT ENO,ENAME,SUM(SALARY) AS TOTALSAL
	FROM EMPLOYEES
	GROUP BY ENO,ENAME
	
	CREATE UNIQUE CLUSTERED INDEX IDX_TOTALSAL
	ON VW_TOTALSAL(ENO)

        Schemabinding:
        =============
        Schemabinding binds the view to the underlying base tables. It prevents changes to the  	table structure that would break the view.
         -- Dropping a column
         -- Altering a table structure


	Note:1) A view can be updated if it is based on single table without Group by,DISTINCT, 	   	Aggregate functions,union,join(In most cases).
             2) Views are not updatable if they contain Group BY,DISTINCT,Aggregate Functions,
                Union,Joins(multiple base tables)
